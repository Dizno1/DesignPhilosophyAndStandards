# Live Regions Pattern

## Why this pattern exists

Live regions can help screen reader users understand changes that happen without a page reload.

They can also become noisy and confusing if overused.

## Use live regions for

- Status changes.
- Completion messages.
- Errors that appear after an action.
- Progress updates when the update affects the user's next decision.

## Avoid live regions for

- Decorative updates.
- Repeating the same message.
- Announcing every small visual change.
- Interrupting audio, practice calls, or other time-sensitive experiences.

## Polite live regions

Use polite announcements for normal status updates.

Examples:

- Saved.
- Upload complete.
- Practice sequence complete.
- Question answered correctly.

## Assertive live regions

Use assertive announcements only when immediate attention is required.

Examples:

- A destructive action failed.
- A session is about to expire.
- A required error prevents continuing.

## Be concise

A useful announcement says enough, then stops.
