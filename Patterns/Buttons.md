# Buttons Pattern

## Why this pattern exists

Buttons perform actions.

Links go places.

Using the right element helps keyboard users, screen reader users, voice control users, and everyone else understand what will happen.

## Use a button when

- The control starts, stops, submits, saves, opens, closes, deletes, plays, pauses, repeats, converts, analyzes, or changes something on the current page.

## Use a link when

- The control navigates to another page, document, section, or external resource.

## Button requirements

- Use a native button element whenever practical.
- Provide a clear visible label.
- Make the accessible name match the visible label whenever practical.
- Ensure the focus indicator is visible.
- Support Enter and Space through native button behavior.
- Do not use disabled controls when a helpful explanation would be better.

## Button labels

Good labels:

- Start Practice
- Stop Practice
- Repeat Last Call
- Save Changes
- Download Report

Weak labels:

- Click Here
- Submit when the action is not clear
- Go when the destination is not clear
- Icon-only controls without a clear name
