# Color Palette

## Purpose

The Open Door Design color palette supports a calm, trustworthy, accessible interface. Color choices must exceed minimum accessibility requirements wherever practical.

## Contrast standard

- Normal text target: 7:1 or higher.
- Large text target: 4.5:1 or higher.
- Interactive controls target: 4.5:1 or higher.
- Icons and meaningful graphical elements target: 4.5:1 or higher.
- Focus indicators target: 4.5:1 or higher against adjacent colors.

## Core colors

### Deep Navy

Hex: `#17324D`

Use for:

- Headings
- Header backgrounds
- Footer backgrounds
- High-emphasis interface areas

### Open Door Green

Hex: `#2E7D32`

Use for:

- Primary buttons
- Positive actions
- Success states
- Active states

### Warm White

Hex: `#FAFAF8`

Use for:

- Page backgrounds
- Large reading areas

### Surface White

Hex: `#FFFFFF`

Use for:

- Cards
- Panels
- Dialogs
- Form groups

### Soft Border Gray

Hex: `#D7D7D7`

Use for:

- Borders
- Dividers
- Card outlines

### Accessible Gold

Hex: `#C58A00`

Use for:

- Focus indicators
- Current state accents
- Progress highlights

Do not use gold for small text on white.

## Text colors

### Primary Text

Hex: `#111111`

Use for body text and most content.

### Secondary Text

Hex: `#3B3B3B`

Use for supporting text.

### Muted Text

Hex: `#5D5D5D`

Use sparingly. Muted text must still meet the contrast standard.

## State colors

### Information

Hex: `#005A9C`

### Warning

Hex: `#8A4F00`

### Error

Hex: `#B3261E`

### Success

Hex: `#2E7D32`

## Color use rules

- Never rely on color alone.
- Pair color with text, icons, borders, labels, spacing, or structure.
- Do not use blue and green together as the primary visual distinction between neighboring interface elements, navigation states, buttons, cards, banners, or important content areas.
- Do not place blue links or blue state indicators directly within a green navigation or green branding system unless there is a clear non-color distinction and the combination has been reviewed for color vision accessibility.
- Use one dominant color family within a single component or region. A banner, navigation bar, card group, or control cluster should not depend on a blue-to-green relationship.
- Avoid low-contrast placeholder text.
- Avoid colored text over images.
- Do not use pale colored buttons with low-contrast text.
- Test colors in forced colors and high contrast modes.
