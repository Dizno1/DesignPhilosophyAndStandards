# Lesson Page Pattern

## Purpose

The lesson page pattern defines the required structure for instructional lessons in Open Door Design Academy applications.

## Required page title

The page title should identify the current domain and lesson. The title should not repeat the site name before the lesson name.

Preferred format:

Domain number and lesson title, followed by the application or course name when needed.

## Required H1

Each lesson page has exactly one H1.

The H1 should identify the current domain and lesson. It should not duplicate the home page course title unless the lesson itself is the home page.

## Required section order

A standard lesson should use this broad order:

1. Lesson overview
2. Related Library Topics when relevant
3. Part controls
4. Lesson content
5. Progress information
6. Navigation controls

## Lesson parts

Lesson parts should be organized as meaningful sections, not arbitrary chunks.

For CPACC-style lessons, the preferred instructional sequence is:

1. Overview
2. Opening scenario
3. Core learning
4. Barrier thinking or lesson-specific analysis
5. Real-world examples
6. Professional perspective
7. Accessibility implications
8. Common misconceptions
9. Exam focus
10. Lesson summary

The last two parts should normally be exam focus and lesson summary when the course is assessment preparation.

## Headings

Headings must support fast screen reader navigation.

Rules:

- One H1 per page.
- H2 for major page sections.
- H3 for lesson parts.
- Do not duplicate heading labels without a purpose.
- Avoid vague headings such as Related, More, or Information when a more descriptive label is available.

## Related Library Topics

Related Library Topics provide optional reference material. They should appear early enough to be discoverable but should not interrupt the main instructional content.

Related topics should be relevant to the current lesson.

## Part controls

Part controls help learners move through a long lesson without losing position.

Controls should be clearly named and keyboard reachable.

Common controls include:

- Previous Part
- Related
- Next Part

Unavailable controls must be conveyed programmatically and visually.

## Progress information

Progress should be useful and concise.

Avoid repeating progress messages after every minor action unless the change is meaningful.

## Screen reader behavior

The screen reader experience should let the learner understand:

- The current lesson
- The current part
- Available navigation controls
- Progress through the lesson
- What changed after activating a control

## Mobile behavior

On small screens, controls should stack vertically, remain full width where helpful, and maintain comfortable touch targets.

## Why this pattern exists

Long lessons are difficult to navigate when structure is inconsistent. This pattern gives learners predictable orientation points so they can concentrate on the content.
