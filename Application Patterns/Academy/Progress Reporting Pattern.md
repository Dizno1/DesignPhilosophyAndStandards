# Progress Reporting Pattern

## Purpose

Progress reporting tells learners where they are and how much work remains.

It should support confidence without becoming noise.

## Progress types

Academy applications may report:

- Course progress
- Domain progress
- Lesson progress
- Position within the current lesson
- Completion state

## Language

Use plain language.

Examples:

- Course Progress: 42 percent complete.
- Domain Progress: 60 percent complete.
- Lesson Position: Part 4 of 10.
- Lesson Completion: Complete.

## Visual and programmatic requirements

Progress must not depend on color alone.

Progress should be available to screen reader users through text, not only through a visual bar.

## Live announcements

Announce progress only when the update is meaningful.

Do not repeatedly announce progress after every small movement if the learner already has enough context.

## Reflow

Progress indicators must work in a narrow single-column layout without horizontal scrolling.

## Why this pattern exists

Progress can reduce anxiety and increase confidence, but excessive announcements can become a barrier. The Academy pattern treats progress as orientation support, not decoration.
