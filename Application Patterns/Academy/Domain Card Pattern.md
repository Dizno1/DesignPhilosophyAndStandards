# Domain Card Pattern

## Purpose

Domain cards introduce major course sections and help learners choose where to go next.

## Required content

A domain card should include:

- Domain title
- Short description
- Lesson count or scope when useful
- Progress or completion state when available
- Clear action link or button

## Link and button rules

Do not nest buttons inside links.

If the card navigates to a domain page, use a link with clear text.

If the card performs an action on the current page, use a button.

## Heading rules

Domain titles should use headings when the card list represents major page content.

Heading levels must fit the surrounding page structure.

## Reflow

Cards should stack cleanly on narrow screens.

No domain card should require horizontal scrolling.

## Screen reader behavior

Each card should read as a coherent group.

The user should understand the domain name, purpose, state, and action without visual layout.

## Why this pattern exists

Domain cards are the learner's doorway into the course. They must be informative, compact, and easy to navigate by heading, link, or touch.
