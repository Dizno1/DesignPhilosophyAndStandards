# Academy Accessibility Requirements

## Baseline

Academy applications must meet Open Door Design accessibility standards and target WCAG 2.2 AA while exceeding minimum requirements where practical.

## Required testing

Academy releases should be tested with:

- Keyboard only
- JAWS with Chrome
- NVDA with Firefox
- VoiceOver on iPhone
- Browser zoom through 400 percent
- Reflow at 320 CSS pixels
- Reduced motion settings
- High contrast or forced colors where practical

## Page titles

Page titles should identify the current page before the application or course name.

## Headings

Each page has exactly one H1.

Lesson pages use the H1 for the current domain and lesson, not the generic course title.

## Keyboard workflow

Common tasks should not require excessive tabbing.

Controls must appear in a logical order.

## Screen reader workflow

The learner should be able to navigate efficiently by headings, buttons, links, and landmarks.

Announcements should be meaningful and not repetitive.

## Focus management

Focus should remain predictable after navigation, part changes, dialogs, and progress updates.

## Reflow

All lesson pages, domain pages, cards, controls, and progress indicators must work without horizontal scrolling at 320 CSS pixels.

## Color and contrast

Academy applications use the Open Door Design color palette and contrast requirements.

Text should target 7 to 1 contrast whenever practical.

Interactive elements should target 4.5 to 1 contrast.

## Why this document exists

Academy applications are content-heavy and navigation-heavy. They need accessibility rules that address real learner workflows, not only individual controls.
