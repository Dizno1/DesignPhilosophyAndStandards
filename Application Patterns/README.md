# Application Patterns

Application patterns document complete Open Door Design application architectures.

A component describes one reusable piece of interface. An application pattern describes how multiple components, standards, and workflows combine into a full product.

Use this folder when a pattern is larger than a button, card, form, dialog, or carousel.

Current application patterns include:

- Academy
- Assessment Lab
- Beepball Practice Caller, planned
- Open Door Design Website, planned

Application patterns should explain:

- Purpose
- Primary users
- Core workflows
- Required page types
- Required components
- Heading and landmark rules
- Navigation behavior
- Screen reader behavior
- Keyboard behavior
- Reflow behavior
- Testing requirements
- Reasons for major design decisions
