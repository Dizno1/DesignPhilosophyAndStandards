# Domain Card Component

## Purpose

The domain card component presents a major course section and links learners to its content.

## Required content

- Domain title
- Short description
- Action link
- Progress state when available

## Accessibility requirements

- The card has a meaningful heading when appropriate.
- Link text describes the destination.
- Buttons are not nested inside links.
- Color is not the only completion indicator.
- Cards stack cleanly at narrow widths.

## Why this component exists

Domain cards help learners understand the course structure quickly and choose a clear next action.
