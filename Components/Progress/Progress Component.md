# Progress Component

## Purpose

The progress component communicates user position and completion state.

## Required text

Progress must be available as text.

Do not rely only on a visual bar.

## Recommended labels

- Course Progress
- Domain Progress
- Lesson Progress
- Lesson Position
- Lesson Completion

## Accessibility requirements

- Text equivalent is always present.
- Color is not the only indicator.
- Updates are announced only when meaningful.
- Progress bars, if used, expose name, value, and range.
- The component reflows without horizontal scrolling.

## Example text

Course Progress: 42 percent complete.

Domain Progress: 60 percent complete.

Lesson Position: Part 4 of 10.

Lesson Completion: In progress.

## Why this component exists

Progress is orientation. It should help users understand where they are without interrupting their task.
