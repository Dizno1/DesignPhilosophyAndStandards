# Accessibility Promise

Open Door Design does not design to the minimum accessibility requirements.

We design software people can trust.

Whenever practical, Open Door Design exceeds WCAG minimum requirements for contrast, keyboard accessibility, screen reader usability, responsive reflow, predictable interaction, error recovery, and plain language.

Accessibility is not a compliance exercise.

Accessibility is the foundation of every design decision.

## What users should be able to expect

Users should be able to:

- Navigate without a mouse.
- Understand where they are.
- Understand what each control does.
- Complete meaningful work with a screen reader.
- Zoom without breaking the page.
- Reflow content without horizontal scrolling during normal workflows.
- Recover from errors without technical knowledge.
- Trust that the interface will not unexpectedly move focus, hide information, or change behavior.
